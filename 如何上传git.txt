首先先将你要上传的仓库从远程导到本地
鼠标右键打开git bash
git -clone 地址
此时你的文件夹会多出一个名字为你的仓库名的一个文件夹，进入该文件夹，重新鼠标右键打开一个git bash，之前的可以关闭了
git status 是查看当前文件夹下的文件状态。由于.git文件夹中有关于每次的改动，因此会有告诉是否存在哪些文件是还没有加入到缓存区的。
第二步
git add -A
将所有的当前文件夹下没有加入缓存区的添加进缓存区
git cimmit -m "本次提交如暂存区的操作名字如first commit"
将所有的缓存区中文件提交到本地仓库
第三步
git push -u origin main 
///origin后面的是分支，可以在.git中的config找到是什么，一般是main，可以更改设置成master.
这是将本地仓库和GitHub的远程仓库联系，之后刷新即可看到上传成功。

上传过程中可能有网络超时问题，time out，没有网络代理vpn的多次提交即可。
有的话可以换上提交。
